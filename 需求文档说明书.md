# 需求规格说明书

## 1. 项目概述
### 1.1 项目背景与业务目标
* **项目背景：** 全国存在大量小型水库，安全隐患突出，传统人工巡查及单一的数据监测无法满足现代防汛抗旱及大坝安全管理要求。本系统旨在利用信息化技术、GIS和物联网，建立一套自动化、智能化的监测预警体系。
* **业务目标：** 实现小型水库雨水情、大坝位移及渗漏等关键数据的实时采集与分析，当数据异常时触发安全预警。目标是提高水库安全管理的效率和准确性。

### 1.2 合规依据
1. 《水利信息系统开发规范与流程文档》（methodology.docx）全章节要求
2. 《水利信息化顶层设计导则》（SL/Z 729-2016）

### 1.3 项目边界
* **功能边界：** 涵盖水库基础信息管理、实时监测、GIS空间展示、预警分析，不包含气象部门的大范围气象预测。
* **用户边界：** 供水务局管理人员、水库管护人员、巡检人员使用。
* **技术边界：** 采用B/S架构，后端基于Python框架，数据库使用PostgreSQL+PostGIS。

### 1.4 术语定义
* **GIS:** 地理信息系统，本项目中用于水库及周边水利空间数据展示、分析的核心技术。
* **WGS84:** 空间数据坐标存储标准（EPSG:4326）。

## 2. 用户与角色权限定义
### 2.1 角色清单
* **系统管理员：** 负责系统配置、用户管理、权限分配等。
* **业务管理员：** 各级水行政主管部门人员，负责水库信息审批、预警处理。
* **巡检操作人员：** 水库日常巡检员，负责数据上报和设备维护。

## 3. 功能需求规格
### 3.1 系统通用管理模块
包含用户管理、角色权限管理、日志审计、字典管理、系统配置，满足安全性与合规性要求。

### 3.2 小型水库安全监测模块
* **水库基础信息管理：** 水库位置、库容、水位高程参数等信息的增删改查。
* **实时监测数据管理：** 接入水位、雨量、大坝表面位移等时序数据。
* **安全预警管理：** 阈值设置、超限报警生成、短信/系统通知分发。

### 3.3 GIS空间分析模块
* **水利要素地图可视化：** 将水库、监测站、汇水区等信息在地图上标注显示。
* **空间查询：** 基于空间范围查询关联水利设施。
* **缓冲区分析：** 溃坝淹没区模拟及缓冲区内的影响范围分析。

### 3.4 数据可视化与统计模块
* **水库安全态势大屏：** 核心指标（水位变化、高风险水库统计）大屏展示。

## 4. 非功能需求规格
### 4.1 性能需求
* 页面响应时间：常规页面加载时间≤2s，空间查询与渲染操作时间≤3s。

### 4.2 兼容性需求
* 适配现代主流浏览器（Chrome 90+，Edge）。

### 4.3 安全性需求
* 无越权操作、SQL注入、XSS等漏洞。敏感数据实施加密存储与传输。

### 4.4 可维护性与可扩展性需求
* 系统架构支持水平扩展，RESTful API需具备良好的版本控制。

## 5. 数据需求规格
### 5.1 空间数据要求
* **坐标系标准：** 存储采用 WGS84（EPSG:4326），Web显示适配 Web Mercator（EPSG:3857）。
* **拓扑与质量要求：** 多边形不得自相交，要素间符合拓扑规则。

### 5.2 数据库设计要求
* **数据库选型：** PostgreSQL + PostGIS。
* **设计要求：** 表名字段采用 `snake_case`；空间字段建 `GIST` 索引；时序表分区；核心表包含 `created_at`、`updated_at`、`is_active`。

## 6. 工程实现约束
必须严守《水利信息系统开发规范与流程文档》，包括项目骨架、命名、设计规范、安全规范等。
