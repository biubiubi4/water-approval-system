## 1. 项目定位与架构

项目用于对取水申请及其附件进行完整性、法规检索和合规性初审。

```text
Vue 3 前端（3000）
        |
        | /api/**
        v
Spring Boot 主后端（8080）
        |
        | AI_SERVICE_BASE_URL，默认 http://127.0.0.1:8000
        v
FastAPI AI 服务（8000）
        |
        +-- 附件解析 / PDF OCR
        +-- Chroma 知识库检索
        +-- 本地确定性规则
        +-- 按需调用 Qwen
```

调用边界已经统一为：`Vue 前端 -> Java 主后端 -> Python AI 服务`。

- 前端只访问 `/api/**`，不应直接访问 Python 的 8000 端口。
- Java 后端负责申请、附件和审核结果的业务编排与持久化，并通过 `AiServiceClient` 调用 Python。
- Python 服务只提供内部智能审核、知识库和附件解析能力。

关键入口：

- 前端：[frontend/vite.config.js](frontend/vite.config.js)，开发代理将 `/api` 转发给 Java 8080。
- Java 客户端：[java-backend/src/main/java/com/waterapproval/service/AiServiceClient.java](java-backend/src/main/java/com/waterapproval/service/AiServiceClient.java)。
- Java 知识库代理：[java-backend/src/main/java/com/waterapproval/controller/KnowledgeController.java](java-backend/src/main/java/com/waterapproval/controller/KnowledgeController.java)。
- Python 审核编排：[python-ai-service/app/agent.py](python-ai-service/app/agent.py)。

### 3.1 规则模块当前包含的检查

- 必填字段：申请人、证件号或统一社会信用代码、项目名称、取水用途、取水地点。
- 必备材料：取水许可申请书、身份证或营业执照为硬性拦截；水资源论证相关材料为警告。
- 填写规则：取水用途选择“其他/其它”时必须提供具体说明。
- 附件内容一致性：如文件名是身份证但正文显示驾驶证，会直接拦截；营业执照、申请书内容异常会产生警告。
- 有效期：从附件文本中识别已过期的证件或资质。
- 表单与附件一致性：申请人名称、证件号未在附件解析文本出现时产生警告。

规则结果统一放在 `details.fast_rules`，结构包含 `blockers`、`warnings`、`suggestions`、`submitted_materials` 等字段。

### 3.2 审核策略

`REVIEW_MODE`（代码配置字段为 `review_mode`）支持三种模式：

| 模式 | 是否调用 Qwen | 适用场景 |
| --- | --- | --- |
| `fast` | 从不调用 | 批量预检、演示、低风险材料 |
| `smart`（默认） | 仅风险信号出现时调用 | 日常业务，推荐 |
| `strict` | 外部 AI 可用时总是调用 | 复杂或人工复核前的严格审查 |

`smart` 模式会在以下情况调用 Qwen：

- 确定性规则出现警告，需要语义复核。
- 检索到含“禁止、不得、限制、不予批准、保护区”等限制性法规。
- 已解析附件数量不少于 4，或累计解析文本达到 1200 字符。

出现硬性规则问题时，会立即返回 `REJECTED`，跳过附件解析和 Qwen；附件解析后新发现硬性规则，也会跳过 Qwen。审核路径和原因可在 `details.ai_review_trace` 中查看：

- `used_external_ai`：是否真正调用并采用了外部 AI。
- `qwen_decision_reason`：为何调用或跳过 Qwen。
- `fallback_to_local_rules` 与 `fallback_reason`：本地规则兜底原因。

## 4. Qwen 调用位置与如何确认

- 文本审核调用点：[python-ai-service/app/llm_client.py](python-ai-service/app/llm_client.py) 的 `generate_review`，模型为 `qwen3.6-plus`（以配置为准）。
- 扫描 PDF/复杂页面 OCR 调用点：[python-ai-service/app/pdf_reader.py](python-ai-service/app/pdf_reader.py)，模型为 `qwen-vl-plus-latest`（以配置为准）。
- Qwen 仅在 Python AI 服务中调用，Java 和 Vue 不直接调用 Qwen。

排查一次审核时，优先检查：

1. Python 控制台的 `[AI审核]`、`[规则审查]` 和 Qwen 调用日志。
2. Java 保存的审核结果中 `details.ai_review_trace`。
3. Python 环境是否配置了外部 AI 所需的 API Key；未配置或调用失败时，系统会回退到本地规则。
