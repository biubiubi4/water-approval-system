# 迁移与检索优化变更说明

## 1. 修改前概况

Python AI 服务的运行时数据位于项目源码目录 `python-ai-service/data/`：

- Chroma 向量库：`data/chroma`。
- 文档解析缓存：`data/document_cache`。
- 知识库上传文件：`data/uploads`。
- 嵌入默认使用 256 维 `HashEmbeddings`；虽然配置中写有 BGE 模型名称，但默认不会加载该模型。

检索流程为多查询变体的 Chroma 向量召回，再以“向量相似度 65% + 简单关键词包含率 35%”排序。它容易遗漏法规名称、条号、地名等精确词项；旧哈希向量对中文语义表达的区分能力也有限。

## 2. 已完成的修改

### 运行时数据迁移

已将 AI 服务运行数据统一配置到 `E:\water-approval-runtime`：

| 内容 | 新位置 |
| --- | --- |
| Chroma 向量库 | `E:\water-approval-runtime\chroma` |
| 文档解析缓存 | `E:\water-approval-runtime\document_cache` |
| 知识库上传文件 | `E:\water-approval-runtime\uploads` |
| BGE / Hugging Face 模型缓存 | `E:\water-approval-runtime\huggingface` |

涉及的代码为：

- `python-ai-service/app/config.py`：运行时目录默认配置。
- `python-ai-service/main.py`、`python-ai-service/app/service.py`：知识库上传、删除及来源文件处理改为使用新上传目录。
- `python-ai-service/app/embeddings.py`：模型加载显式使用新的模型缓存目录。

原 `python-ai-service/data/` 中的文档缓存和上传文件已迁移。旧 Chroma 索引及其备份按要求删除，因此正式向量库目前为空。

### 检索精度优化

- 默认嵌入提供方从 `hash` 改为 `sentence_transformers`。
- 默认嵌入模型为 `BAAI/bge-small-zh-v1.5`，模型已缓存到 E 盘运行时目录，实测输出维度为 512。
- 向量候选数量由 12 增至 24。
- 增加 BM25 稀疏召回：在本地知识库中补召回包含法规名称、条号、地名等精确术语的片段。
- 最终融合排序更新为：向量相似度 60% + 关键词覆盖 20% + BM25 20%。接口结果新增 `bm25_score` 字段。

相关实现位于 `python-ai-service/app/tools.py`。

## 3. 修改原因

1. **运行数据与源码分离**：避免向量库、缓存、上传文件及模型占用项目工作区，便于备份、清理和迁移。
2. **避免旧索引不兼容**：Hash Embedding 与 BGE 生成的向量不能混用，因此删除旧 Chroma 数据并要求重新建库。
3. **提升中文语义召回**：BGE 能理解相近的中文表达，替代哈希特征的机械匹配。
4. **补足精确匹配**：BM25 能改善法规名称、条号与地点等词汇在纯向量检索中被漏召回的情况。

## 4. 已删除的废弃内容

- 旧 Chroma 向量库及其备份。
- 迁移后为空的 `python-ai-service/data/document_cache`、`python-ai-service/data/uploads` 及父目录 `python-ai-service/data`。
- 无代码引用的 `runtime_dir`、`embedding_fallback_provider` 配置项。
- `.gitignore` 中旧的 `python-ai-service/data/document_cache/` 规则。
- 部署和设计文档中已失效的旧运行时路径说明已更新。

## 5. 后续使用与注意事项

1. 从知识库管理页重新上传法规、规范等材料，建立新的 BGE 向量索引。
2. 运行环境需要安装 `python-ai-service/requirements.txt`；本机已完成安装。
3. 首次加载 BGE 会使用 `E:\water-approval-runtime\huggingface` 中的模型缓存。Windows 未开启开发者模式时可能显示符号链接性能提示，不影响模型正常使用。
4. BM25 当前对 Chroma 中的本地文档做内存计算，适用于本项目的中小规模法规库；若后续知识库规模大幅增加，应改为独立全文检索引擎或预构建倒排索引。
