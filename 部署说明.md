# 部署说明

本文档说明涉水审批智能审核系统的本地部署与生产化部署方式。系统由三部分组成：前端、Java 主后端、Python AI 服务。

## 一、环境准备

### 1.1 基础环境

- Windows / Linux / macOS 均可
- Java 17
- Maven 3.8+
- Python 3.9+
- Node.js 18+
- MySQL 8+

### 1.2 代码目录

- 前端：`frontend`
- Java 后端：`java-backend`
- Python AI 服务：`python-ai-service`

## 二、配置项

### 2.1 Java 后端

Java 后端读取以下配置：

- `SPRING_DATASOURCE_URL`
- `spring.datasource.username`
- `spring.datasource.password`
- `AI_SERVICE_BASE_URL`

默认数据库连接指向本机 MySQL：`jdbc:mysql://localhost:3306/water_approval_db`

### 2.2 Python AI 服务

Python 服务默认使用本地配置，无需强制外部模型即可启动。

可选项包括：

- 外部模型开关
- 外部模型 API Key
- 外部模型 Base URL
- 外部模型名称

默认向量库目录位于 `E:\water-approval-runtime\chroma`

## 三、本地部署步骤

### 3.1 启动 MySQL

先确保本地 MySQL 已启动，并创建数据库：

```sql
CREATE DATABASE water_approval_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

若账号密码不是默认值，请同步调整 Java 后端配置。

### 3.2 启动 Python AI 服务

```bash
cd python-ai-service
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

启动后访问：

- `http://127.0.0.1:8000/health`
- `http://127.0.0.1:8000/docs`

### 3.3 启动 Java 后端

```bash
cd java-backend
mvn spring-boot:run
```

启动后访问：

- `http://localhost:8080`

### 3.4 启动前端

```bash
cd frontend
npm install
npm run dev
```

启动后访问：

- `http://localhost:3000`

## 四、推荐启动顺序

1. 启动 MySQL
2. 启动 Python AI 服务
3. 启动 Java 后端
4. 启动前端

原因是 Java 后端会调用 Python AI 服务，而 Python 服务会初始化知识库和向量存储。

## 五、生产部署建议

### 5.1 Java 后端

```bash
cd java-backend
mvn clean package
java -jar target/water-approval-backend-0.1.0.jar
```

如需修改端口或数据库连接，建议通过环境变量注入，而不是直接改配置文件。

### 5.2 Python AI 服务

```bash
cd python-ai-service
pip install -r requirements.txt
python main.py
```

如需守护进程运行，可在 Linux 下配合 systemd 或 supervisor，在 Windows 下配合计划任务或 NSSM。

### 5.3 前端静态部署

```bash
cd frontend
npm install
npm run build
```

构建产物位于 `frontend/dist`，可交由 Nginx、Apache 或任意静态站点托管。

## 六、验证步骤

部署完成后建议按以下顺序验证：

- `GET /health` 检查 Python 服务
- `GET /api/ai/tools` 检查 Java 到 Python 的转发链路
- `GET /api/applications` 检查 Java 后端是否正常
- 打开前端首页检查页面能否加载

## 七、常见问题

### 7.1 Java 后端连不上 AI 服务

检查 `AI_SERVICE_BASE_URL` 是否正确，默认应指向 `http://127.0.0.1:8000`

### 7.2 Python 服务启动后知识库为空

确认 `E:\water-approval-runtime\chroma` 是否可写；如果向量库损坏，可删除该目录后重启服务重新初始化。

### 7.3 前端无法访问后端接口

确认前端运行时使用的 API 地址与 Java 后端地址一致，且浏览器未拦截跨域请求。
