# 运行时迁移与检索优化记录

## 运行时目录

已将 Python AI 服务的可变运行数据统一配置到 `E:\water-approval-runtime`：

- `chroma`：Chroma 向量数据库。
- `document_cache`：PDF、Word、OCR 等文档解析缓存。
- `uploads`：知识库上传文件。
- `huggingface`：SentenceTransformer / Hugging Face 模型缓存；BGE 首次使用会下载到这里。

`app/config.py` 中默认启用 `chroma_dir`、`document_cache_dir`、`uploads_dir`、`model_cache_dir`。这些配置均可用同名大写环境变量覆盖，例如 `CHROMA_DIR`。

旧 Chroma 索引已按要求清除，必须重新上传法规或规范后才会产生检索结果。旧文档解析缓存和知识库上传文件已迁移到新的运行时目录。

## 检索优化

- 默认嵌入由 256 维 Hash Embedding 改为 `sentence_transformers` 的 `BAAI/bge-small-zh-v1.5`，提升中文语义匹配能力。
- 模型加载显式使用 `model_cache_dir`，避免写入用户 C 盘缓存。
- 向量候选数从 12 提升到 24。
- 在原有多查询变体、关键词覆盖重排基础上，加入本地 BM25 稀疏召回，补足法规名称、条号、地名等精确匹配。
- 最终分数为：向量相似度 60% + 关键词覆盖 20% + BM25 20%。返回结果新增 `bm25_score` 字段。

## 使用说明

1. 安装 `python-ai-service/requirements.txt` 中的依赖，并确保机器可下载或已预置 BGE 模型。
2. 启动 AI 服务后，从知识库管理页重新上传法规、规范和示例材料，建立新的 BGE 向量索引。
3. 若无网络，可预先将模型放到 `E:\water-approval-runtime\huggingface`，或临时设置 `EMBEDDING_PROVIDER=hash`（精度会明显下降）。
